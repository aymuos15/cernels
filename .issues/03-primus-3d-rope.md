# 03 · 3D axial RoPE (nnU-Net Primus)

**What it is.** Primus (MIC-DKFZ, "Enforcing Attention Usage for 3D Medical Image Segmentation") is a Transformer-centric 3D segmentation backbone. It tokenizes a volume into a 3D grid of patches and runs an EVA-02-style Transformer over the flattened token sequence, with **3D axial rotary position embedding** applied to q/k in every attention block. The implementation lives in `dynamic_network_architectures/building_blocks/eva.py` (class `Eva`) and is built directly on **timm**: `RotaryEmbeddingCat` (from `timm.layers`) builds the rotary frequencies over an N-d feature grid, and timm's `apply_rot_embed_cat(x, rope)` does the rotation inside `EvaBlock`/`EvaAttention` (from `timm.models.eva`). The "3D" part is feeding `RotaryEmbeddingCat` a **3-tuple** `ref_feat_shape = input_shape // patch_embed_size` so the frequencies are split across the depth/height/width axes (axial RoPE). Per the paper, RoPE is crucial for Primus — it improves accuracy and training stability (KiTS23, LiTS).

**Baseline — library reference, NOT hand-written.** Per [setting up baselines](../../docs/guide/setting_up_baselines.md), the baseline is timm's own functions run directly: build the embed once with `RotaryEmbeddingCat(dim=rope_dim, in_pixels=False, feat_shape=grid, ref_feat_shape=grid).get_embed()`, then `baseline` = `apply_rot_embed_cat(q, rope)` / `apply_rot_embed_cat(k, rope)`. No reimplementation. **Setup:** timm is not yet a dependency — add it to the `benchmark` extra (`uv add --optional benchmark timm`) and import it lazily in the config. There is no Hub kernel for 3D RoPE, so `lib` is unset; the contender is a `custom` fused CUDA kernel.

**Config sketch.** `Config` (non-Hub), dtype bf16 (Primus trains in mixed precision; rotation accumulate in fp32 in the kernel). `op = "timm.layers.apply_rot_embed_cat"`. `inputs(device, dtype)` builds raw `q`, `k` of shape `(B, n_heads, N, head_dim)` plus the **precomputed** rope embed (cos/sin cat) — the embed is built once per fixed grid and cached/shared across all blocks in the real model (exactly like cos/sin are inputs to our 2D `rotary` config), so precomputing it in `inputs()` is faithful; the rotation is the distinctive timed work. `baseline` applies timm's rotate to q and k. `custom` = one fused CUDA pass over q and k (mirror `src/kops/registry/rope.cu`, but for the cat-style/axial layout rather than the duplicated-half llama layout). `verify` at atol ~2e-2 (bf16).

**Inputs to think about.** Primus-M shape: `embed_dim=792`, `n_heads=12` → `head_dim=66`, `rope_dim = head_dim/1.5 = 44` (timm asserts `rope_dim % 4 == 0`). 3D token grid `20×20×20 = 8000` tokens (8³ patches over a 160³ crop), batch 2: `q,k = (2, 12, 8000, 66)`. Note only the first `rope_dim` channels per head are rotated (the cat layout), the rest pass through — match timm's exact channel split.

**Difficulty.** Medium. The math is rope (cheap, bandwidth-bound), so — like our 2D `rotary` result (custom 5.06× vs compile 4.91×, i.e. custom barely edges compile) — a **standalone** 3D-rope apply will likely only modestly beat `torch.compile`; compile already does well on elementwise rotate. The honest high-leverage variant is **fusing rope into the attention QK path** (see the existing `fused_qknorm_rope_attention` direction), which removes the q/k round-trip to HBM entirely. Start with the standalone apply to establish the baseline, then consider the fused-attention version. Keep the rotation in the timed path; only the (legitimately cached) rope embed is precomputed in `inputs()`.

**Refs.** Primus paper: https://arxiv.org/abs/2503.01835 · code: `dynamic_network_architectures/building_blocks/eva.py` (`Eva`, uses `RotaryEmbeddingCat`) and `architectures/primus.py` · timm RoPE: `timm.layers.pos_embed_sincos` (`RotaryEmbeddingCat`, `apply_rot_embed_cat`). See our `rotary` config + `src/kops/registry/rope.cu` for the 2D precedent.
